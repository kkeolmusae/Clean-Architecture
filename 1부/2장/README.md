# 2장 두 가지 가치에 대한 이야기

모든 소프트웨어 시스템은 이해관계자에게 서로 다른 두 가지 가치인 행위(behavior)와 구조(structure)를 제공한다.

</br>

## 행위

프로그래머를 고용하는 이유는 이해관계자를 위해 기계가 수익을 창출하거나 비용을 절약하도록 만들기 위해서이다. 많은 프로그래머가 이러한 요구사항을 만족하도록 코드를 작성하는 것이 자신이 해야할 일의 전부라고 생각한다. 물론 틀린 생각이다.

</br>

## 아키텍처

소프트웨어는 Soft + Ware로 부드로운 제품이라는 뜻으로 기계의 행위를 쉽게 변경할 수 있도록 하기 위한다는 뜻이다. 

소프트웨어가 가진 본연의 목적을 추구하려면 변경하기 쉬워야 한다. 아키텍처는 형태에 독립적이어야 새로운 기능을 추가할 때 비용이 적게든다.

</br>

## 결론

행위와 구조 둘다 중요하다. **아키텍처를 후순위에 두면 시스템을 개발하는 비용이 더 많이 들고, 일부 또는 전체 시스템에 변경을 가하는 일이 현실적으로 불가능해진다는 문제가 발생한다.**
